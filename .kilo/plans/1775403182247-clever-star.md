## Stap 2: Docker setup (F1-02)

### Doel
Functionele Docker setup met hot-reload voor frontend én backend.

### Taken

**2.1 Frontend Dockerfile (development)**
```dockerfile
FROM node:20-alpine
WORKDIR /app
COPY package*.json ./
RUN npm install
COPY . .
EXPOSE 5173
CMD ["npm", "run", "dev", "--", "--host"]
```

**2.2 Backend Dockerfile (development)**
```dockerfile
FROM python:3.12-slim
WORKDIR /app
RUN pip install uv
COPY pyproject.toml uv.lock ./
RUN uv sync --frozen
COPY . .
EXPOSE 8000
CMD ["uvicorn", "app.main:app", "--host", "0.0.0.0", "--reload"]
```

**2.3 docker-compose.yml**
- Services: frontend (5173), backend (8000), db (5432)
- PostgreSQL met named volume
- Backend depends_on db
- Health check op backend `/health`

**2.4 Health endpoint implementeren**
- `GET /health` → `{"status": "ok"}`
- Database connectivity check

### Verwachte resultaten
- `docker compose up` start alle services
- Hot-reload werkt bij code wijzigingen
- PostgreSQL data persistent na herstart

## Stap 3: Alembic migraties (F1-03)

### Doel
Versiebeheerd database schema met alle core tabellen.

### Taken

**3.1 Alembic initialisatie in backend**
```bash
cd backend && uv run alembic init alembic
```

**3.2 Configureer env.py**
- DATABASE_URL uit environment
- Target metadata van models

**3.3 Base migration aanmaken**
- Alle tabellen uit `/datamodel/01-core-schema.md`:
  - clubs
  - users
  - invite_tokens
  - banen
  - competities
  - teams
  - speelrondes
  - baantoewijzingen
  - planninghistorie

**3.4 SQLAlchemy models**
- `/backend/app/models/` met alle entity classes
- Base declarative class in `/backend/app/models/base.py`

**3.5 Test run**
- `docker compose up` met db
- `docker compose exec backend alembic upgrade head`
- `docker compose exec backend alembic downgrade -1`

### Verwachte resultaten
- `alembic upgrade head` succesvol
- Alle tabellen aangemaakt in PostgreSQL
- Downgrade werkt correct
