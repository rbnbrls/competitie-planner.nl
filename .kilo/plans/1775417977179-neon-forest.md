# Plan: Admin Onboarding Flow

## Doel
Implementeer een onboarding flow voor admin.competitie-planner.nl zodat de eerste gebruiker zich kan registeren via de frontend zonder CLI/terminal.

## Huidige situatie
- Superadmin login via `/frontend/src/pages/Login.tsx`
- Admin moet via CLI worden aangemaakt (`backend/app/cli/superadmin.py`)
- Geen endpoint om te controleren of admin bestaat

## Implementatie stappen

### Stap 1: Backend - Admin exists endpoint
**Bestand:** `backend/app/routers/auth.py`

- Nieuwe endpoint: `GET /api/v1/auth/admin-exists`
- Checkt of er superadmin gebruikers zijn in de database
- Retourneert `{ "exists": boolean }`

### Stap 2: Backend - Admin registratie endpoint
**Bestand:** `backend/app/routers/auth.py`

- Nieuwe endpoint: `POST /api/v1/auth/register-admin`
- Valideert email formaat en wachtwoord sterkte
- Maakt nieuwe superadmin gebruiker aan (alleen toegestaan als geen admin bestaat)
- Retourneert tokens en redirect info

### Stap 3: Frontend - API integratie
**Bestand:** `frontend/src/lib/api.ts`

- `authApi.adminExists()` - check of admin bestaat
- `authApi.registerAdmin(email, password)` - registreer eerste admin

### Stap 4: Frontend - Registratie pagina
**Bestand:** `frontend/src/pages/admin/RegisterAdmin.tsx` (nieuw)

- Welkomstbericht
- Registratieformulier: email, wachtwoord, wachtwoord bevestigen
- Real-time validatie (email formaat, wachtwoord sterkte)
- Foutmeldingen

### Stap 5: Frontend - App routing
**Bestand:** `frontend/src/App.tsx`

- Bij laden app: check `adminExists` endpoint
- Als geen admin → toon registratie pagina
- Als admin bestaat → toon bestaande login pagina
- Na succesvolle registratie → redirect naar dashboard

### Stap 6: AuthContext update
**Bestand:** `frontend/src/contexts/AuthContext.tsx`

- Toevoegen `register` functie voor admin registratie
- Na registratie automatisch inloggen

## Edge cases
- Race condition: meerdere gebruikers registreren tegelijkertijd → backend handle dit
- Wachtwoord vereisten: min 8 tekens, letters + cijfers
- Rate limiting: later toevoegen indien nodig
